# DNS Wizard v1.3.0 Enhancement Spec

**Goal:** Transform DNS Wizard from a 6-tool utility into a comprehensive network dashboard with 3 new tools, an overhauled home panel, and polished UI throughout.

**Architecture:** Dashboard-first approach — redesign the Health panel into a Network Dashboard with sparkline history and overall health grade, then add Port Scanner, Wi-Fi Scanner, and Network Info as new sidebar tools, then apply UI polish (empty states, transitions, loading states, result animations, toast notifications) across all panels.

**Tech Stack:** Tauri 2, React 18, TypeScript, Rust (tokio, CoreWLAN via Swift bridge for Wi-Fi), CSS custom properties

---

## 1. Network Dashboard (Health Panel Overhaul)

### 1.1 Overall Health Indicator

Replace the current 3-card layout with:

- **Circular status indicator** (120×120px SVG ring): green/yellow/red based on worst status among DNS, Speed, Security. In the center: speed test letter grade (A+ through F) if available, or "—" if no test run.
- **One-line summary** below the ring: contextual text like "Your network looks great" (all good), "DNS needs attention" (DNS issue), "Run a speed test to see your grade" (no data yet).

Color mapping:
- Green: all 3 statuses are "good"
- Yellow: any "warn"
- Red: any "bad"
- Gray: any "unknown"

### 1.2 History Sparkline

- Horizontal bar chart showing last 20 speed test quality scores (0-100) as vertical bars
- Each bar colored by grade: A+/A green, B light green, C/D yellow, F red
- Tap/hover a bar to see: timestamp, grade, download speed, latency
- Below the chart: min/avg/max labels
- Data source: existing `localStorage` speed history (already stored)
- If no history: show placeholder text "Run a speed test to start tracking"

### 1.3 Status Cards (Redesigned)

Same 3 cards (DNS, Speed, Security) but enhanced:
- **Icon** (existing emoji) + **status dot** (existing) + **status text** + **trend arrow** (↑ improved, ↓ degraded, — stable based on comparing last 2 speed tests)
- **Detail text** (existing, responsive to simple mode)
- **Fix button** (existing)
- Card height: auto (currently ~200px max)

### 1.4 Quick Actions

Keep existing row but restyle:
- Primary: "Quick Fix DNS" → accent button
- Secondary: "Run Speed Test", "DNS Leak Test" → outline buttons

### 1.5 Auto-Refresh

- Health data refreshes when panel becomes active (not just on mount)
- After speed test or leak test completes, dispatch a custom event that HealthPanel listens for and re-fetches DNS status

### 1.6 Metric Tiles

Keep the 4 metric tiles (Latency, Download, Jitter, Loss) below the status cards, exactly as current.

## 2. Port Scanner Tool

### 2.1 UI

- **Header**: "Port Scanner"
- **Input row**: Host field + port range field (e.g. `1-1000` or `80,443,8080`) + preset dropdown ("Common Web", "Common Mail", "Full Scan 1-1024", "Custom")
- **Results**: Table with columns: Port, Status (Open/Closed/Filtered), Service, Latency
- **Summary tiles**: Open count, Closed count, Scanned count
- **Export**: CSV/JSON like other tools

### 2.2 Backend (Rust)

New file: `src-tauri/src/port_scan.rs`

- `run_port_scan(host: String, ports: Vec<u16>)` — async TCP connect to each port with 2s timeout
- Uses `tokio::net::TcpStream::connect_timeout` for each port
- Well-known port service names from a hardcoded map (80→HTTP, 443→HTTPS, etc.)
- Progress events via Tauri emit: `port-progress` with `{ port, status }`
- Cancel via AtomicBool
- Max 500 ports per scan to prevent abuse

### 2.3 Frontend Components

- `PortScanPanel.tsx` — follows same pattern as PingPanel
- New `ActiveTool` value: `"ports"`
- Sidebar icon: `Shield` from lucide-react (or `Scan`)
- Keyboard shortcut: Cmd+7

## 3. Wi-Fi Scanner Tool

### 3.1 UI

- **Header**: "Wi-Fi Scanner"
- **Scan button** (scans once, not continuous)
- **Results**: List of networks sorted by signal strength
  - Each row: SSID, signal bars (4 levels), security badge (WPA2/WPA3/Open), channel, band (2.4/5 GHz)
  - Highlight currently connected network
- **Summary tiles**: Networks found, Current network name, Signal strength
- **Export**: CSV/JSON

### 3.2 Backend (Rust + Swift)

New files:
- `src-tauri/src/wifi_scan.rs` — Rust command handler
- `src-tauri/WifiScanner.swift` — Swift helper using CoreWLAN

Swift code:
```swift
import CoreWLAN

class WifiScanner: NSObject {
    func scan() -> [[String: Any]] {
        let client = CWWiFiClient.shared()
        let interfaces = client.interfaces() ?? []
        // For each interface, scan for networks
        // Return: ssid, bssid, rssi, security, channel, band
    }
}
```

Rust calls Swift via Tauri's shell plugin or generates a static library via `swift-rs`.

Fallback on non-macOS: show "Wi-Fi scanning is only available on macOS" message.

### 3.3 Frontend Components

- `WifiPanel.tsx`
- New `ActiveTool` value: `"wifi"`
- Sidebar icon: `Wifi` from lucide-react
- Keyboard shortcut: Cmd+8

## 4. Network Info Tool

### 4.1 UI

- **Header**: "Network Info"
- **Auto-loads on open** (no scan button needed)
- **Sections**:
  - **External**: IP address, ISP, City, Country (from existing `get_public_ip`)
  - **Interface**: Active interface name, connection type (Wi-Fi/Ethernet), MAC address, local IP, gateway
  - **DNS**: Current DNS servers (from existing `get_current_dns`), whether using DHCP or manual
  - **Copy buttons** next to each value for easy clipboard copy
- **Refresh button** to re-scan
- Simple mode: shows fewer details (IP, ISP, DNS status only)

### 4.2 Backend (Rust)

New file: `src-tauri/src/network_info.rs`

- `get_network_info()` — returns a struct with:
  - `public_ip`, `isp`, `city`, `country` (from existing `get_public_ip`)
  - `interface_name`, `connection_type`, `mac_address`, `local_ip`, `gateway`
  - `dns_servers`, `dhcp_mode` (manual/automatic)
- Uses `networksetup` commands and `route -n get default` (already in `sys_config.rs`)
- Gateway from: parsing `netstat -rn` or `route -n get default`

### 4.3 Frontend Components

- `NetworkInfoPanel.tsx`
- New `ActiveTool` value: `"info"`
- Sidebar icon: `Globe` from lucide-react (or `Network`)
- Keyboard shortcut: Cmd+0 (or shift the others)

## 5. UI Polish

### 5.1 Empty States

Each panel gets an illustrated empty state:

- **Speed**: Speedometer icon + "Press Start Test to measure your connection"
- **Ping**: Crosshair icon + "Enter a host and run ping to see results"
- **Leak**: Shield icon + "Apply a DNS profile first to detect leaks"
- **Ports**: Scan icon + "Enter a host and port range to scan"
- **Wi-Fi**: Antenna icon + "Scan for nearby Wi-Fi networks"
- **Network Info**: Loads automatically, empty state only on error: "Could not detect network info"

Implementation: Create a shared `EmptyState` component with icon, heading, description props.

### 5.2 Panel Transitions

- Add CSS `transition: opacity 0.15s ease` on `.app-content`
- When switching tools, fade content out (opacity 0) → swap → fade back in (opacity 1)
- Use a simple state-driven approach: `transitioning` state that adds/removes a CSS class
- DNS wizard step transitions: slide step content in from the right

### 5.3 Loading States

- Standardize loading animation: use the existing `.dns-benchmark-spinner` (border spin) across all panels
- Port Scanner: show "Scanning port X of Y..." with progress bar
- Wi-Fi Scanner: show "Scanning nearby networks..." with spinner
- Network Info: show "Detecting network..." with spinner (brief, auto-loads)

### 5.4 Result Animations

- Speed test grade reveal: already animated (SpeedGauge SVG arc), keep as-is
- DNS benchmark results: stagger entries in with 100ms delay each (`animation-delay` on each row)
- Quick Fix result: pulse the "Apply" button with a brief scale animation (CSS `@keyframes pulse-once`)
- Port scan results: rows appear as each port completes

### 5.5 Toast Notifications

New component: `ToastProvider` + `useToast` hook

- Positioned top-right, stacked, auto-dismiss after 3 seconds
- Types: success (green), error (red), info (blue accent)
- Uses:
  - "DNS applied successfully" (success)
  - "DNS restored to automatic" (success)
  - "DNS cache flushed" (success)
  - "Speed test result saved" (info)
  - "Authorization cancelled" (error)
  - Port scan/leak test completion notices

Implementation: React context-based, rendered in `App.tsx` inside `ThemeProvider`.
